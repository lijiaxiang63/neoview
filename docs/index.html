Tri-planar navigation

Three planes, one crosshair.

Planes XY, XZ, and YZ stay locked to a shared crosshair. Click or drag in any view to move it — the other two follow live, so you always know exactly where you are in the volume.

  • Wheel to scrub — scroll over any view to step slice-by-slice along its axis
  • Spacing-correct aspect — each plane letterboxes to true physical proportions
  • Cursor readout — voxel indices, world coordinates via the affine, and raw / scaled intensity in the status bar
  • Double-click to maximize any pane; the others stay warm behind it

GPU volume rendering

Raycast the whole volume.

The fourth cell is a WebGL2 raycaster — a fullscreen shader that marches rays through a 3D texture. No mesh, no external library, just the volume and the GPU.

  • MIP & composite modes with adjustable density and brightness
  • Orbit / dolly camera — drag to rotate, wheel to zoom, double-click to reset
  • Interactive fidelity — renders at reduced steps while you move, then settles to full quality
  • Big-volume aware — oversized volumes stride-downsample to a texture budget while slice views stay full resolution

Overlay layers

Stack what you need to compare.

Drop additional volumes onto a loaded base to build an ordered stack of layers. Each aligns through its own affine with nearest-neighbor sampling, so differing grids just work.

  • Value maps — warm / cool / diverging colormaps with a threshold window
  • Binary masks — a single flat color at your chosen opacity
  • Label volumes — a distinct color per id, from a golden-angle palette
  • Per-layer visibility, opacity, and kind — reorder freely; slice views only

Region segmentation

Draw a box. Grow a region.

Draw a box on any slice and let one hysteresis + 3D connected-component engine do the work — the same core drives both threshold and grow methods, previewing live as you tune.

  • Threshold or grow — auto-threshold via a plateau-aware Otsu, or seed from inside and flood to a boundary
  • 6- or 26-connectivity, minimum-size pruning, and a safety cap on whole-volume floods
  • Live histogram of the box, with a marker at each threshold
  • Editable & re-editable regions — right-click to reopen; export remaps to a clean label volume

Under the hood

Written from scratch, on purpose.

No parsing, math, or rendering libraries. The whole pipeline — decompress, parse, normalize, render — is hand-written and unit-tested, and every byte of heavy work happens off the main thread.

01 — parse

Both endiannesses, every datatype

A hand-rolled header reader handles all common voxel datatypes, value scaling, quaternion & matrix affines, and a zero-copy typed-array view when the layout allows.

02 — decompress

Streaming gzip, preallocated

Compressed files decompress through a streaming path that preallocates from the gzip size trailer, falling back to chunk accumulation when the trailer lies.

03 — threads

A worker per file

Decompress, parse, stats, and texture packing run in a worker and transfer back as Transferables. The main thread pays only the GPU upload.

04 — stats

Exact percentiles, one pass

Small integer types get an exact 65,536-bin counting pass; floats use a min/max pass plus a histogram. Auto display range picks 2–98 percentile.

05 — texture

Half-float, fused

Sampling, scaling, [0,1] normalization, and half-float packing fuse into one loop, so slider drags stay pure uniform changes — texture memory is never touched.

06 — resilience

Survives context loss

Rendering is dirty-flag rAF — never a continuous loop — and the raycaster rebuilds itself from cached data if the GL context is lost and restored.

And the details

Everything else you'd reach for.

Drag & drop

Drop a .nii / .nii.gz anywhere on the window, or open with Cmd/Ctrl+O.

Display range

Dual-thumb slider with Auto (2–98 pct), Full range, and fixed presets — an apt one is chosen on load.

4D volumes

A frame slider appears automatically for volumes carrying a fourth dimension.

Affine panel

The full 4×4 voxel-to-world matrix, dimensions, spacing, datatype, and which transform source the file provided.

Export regions

Segmented regions serialize back out to a clean label volume — collision-suffixed, with reveal-in-file-manager.

Cross-platform

Packages to macOS (dmg), Windows (nsis), and Linux (AppImage + deb) from one codebase.
